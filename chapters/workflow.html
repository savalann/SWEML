Workflow Management / Cloud Computing#

Workflow Management: GeoWeaver#

The model has been adapted to the open-source workflow management sytem, GeoWeaver. Geoweaver is a web-based application for interactive, full-stack machine leanring workflow management. The app provdies a user-friendly GUI to interact with and persistently store script logs and code revision histories. Shell and python scripts are supported by the platform, as well as seammless intergration wth Jupyter.

To install and open a Geoweaver instance perform the following steps;

  1. Download the latest release of geoweaver.jar.

  2. Path to the download folder containing the .jar file

  3. Run the command, java -jar geoweaver.jar

  4. Open a browser and enter: http://localhost:8070/Geoweaver/

You will now see the Geoweaver front page. Here you can create, import, edit, and execute processes and build workflows. Please refer to the GEOWEAVER documentation for a full overivew of capabilities.

The National Snow Model has been adpated to execute in Geoweaver. A full-stack workflow of the model can be found at the github page, here. After downloading the zip folder, import the JSON workflow file to Geoweaver using the “import wrokflow” tool in Weaver and the model will be displayed in the weaver window. This workflow is set up to execute from the local machine.

For simplicity, the National Snow Model workflow is reduced to a single sub-region in Colorado. Each node in the workflow has been assigned to a stage in the model devlopment and exectuition process, from training to interactive SWE mapping. Python scripts of each node may be interacted with under the Processes header. Any edits to processes made here will be relected in the Weaver node and a history of revisions saved in Geoweaver.

Training stages of the model can take a long time to execute on a personal computer. The donwloaded model has already undergone trianing, so these steps are not neccessary to perform for the prediction stages of the model. A node may be skipped by double-clicking and selecting “skip”. Before exectuion it is necessary to update the line below in the Folder_Path process to point to the root folder containing the data and models. os.chdir(r”C:\Users\My Name\Desktop\Geoweaver_ML_SWE”)

Cloud Computing: Binder#

Running tutorial workflows can be helpful for new users to toubleshoot during model development. Binder is a free hosting site where users can run their models and data processing workflows for free on the cloud. We have made effort to run the scripts on Binder, however, there are many limitations that challenge the operation of large workflows such as this. For example, there are memory limitations and only 2 CPUs supported on Binder. Further more, Binder uses python 3.7, which can cause some campatibility issues between packages.
