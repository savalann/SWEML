Reproducibility#

Reproducibility is essential for open science. Want to run the tutorial on your own machine? Follow these steps for using git clone. We recommend creating a virtual environment for installing all packages. Some packages are troublesome to install and/or have dependency requirements, please look to the requirement.txt file for more information on specific versions. The UCI version archives are an excellent source of the versioned binaries (.whl files). Binaries are easy to install and here is a guide.

Incorporating new data sources.#

Data driven modeling requires large quantities of quality data. Snow science is no exception and we are constantly looking for more high spatial and temporal data. For example, we are looking to integrate Modis snow products such as snow-covered-area (SCA) and use LandSat land surface imagery to describe vegetation cover.

Modis SCA data could help inform the model if there is snow covering the 1-km grid or not. This could enhance early and late season predictions. Landsat data could help inform the model on the dominant vegetation influence on snowpack. For instance, aspen groves allow much more solar radiation to influence snow process than evergreen forests. Likewise for forested areas vs. open meadows.

Science Questions#

There are numerous questions stemming from the model. Below are a few key questions to help advance our modeling efforts:

  1. What is optimial spatial resolution for SWE modeling for streamflow prediction, taking into account computational limitations for operational hydrology.

  2. Can the submodeling domain be optimized? If so, how and what the modeling performance expectaitons?

  3. Is there a more efficient and logical development of submodels? For instance, can we develop a method that uses the nearest in situ observation location to extrapolate SWE inferences for CONUS, and thus, negate the need for submodels?

  4. How can snow process physics be incorporated in the model, i.e., water and energy budgets and exchanges?
